- Format: Print, web, both? → **Decision: Format-agnostic plain text**
- Integration: Adapt structure for creative writing? → **Decision: Yes, adapted from code templates**

**Decision rules applied:**
- Used most common forms (sonnet, haiku, free verse)
- Targeted intermediate knowledge level
- Covered core creation process only
- Adapted structure for non-code content

### Phase 3: Structure ✓

**Sections defined:**

```

CORE SECTIONS:
1. Poetry Forms - Traditional and contemporary forms with examples
2. Core Techniques - Imagery, metaphor, sound, line breaks
3. Revision Guidelines - Process and checklist
4. Theme and Meaning - Show vs tell, earned emotion
5. Tone and Voice - Consistency and voice choices
6. Structure and White Space - Stanzas, spacing, titles
7. Diction and Word Choice - Precision, active verbs

ANTI-PATTERNS (12 identified):
- Using clichés
- Forcing rhyme at expense of meaning
- Telling emotions directly
- Using "poetic" language unnaturally
- Ignoring line breaks
- Over-explaining
- Adding filler words
- Using only abstractions
- Mixing metaphors
- Relying on archaic language
- Writing about writing
- Default past tense narration

KEY DECISIONS:
- Forms: Include 6 major types
- Examples: Show both good and bad
- Revision: Multiple-pass approach
- Voice: Authentic over decorated
```

## Problems Avoided

Thanks to AGENTS_TEMPLATE.md guidance:

### Problem: Too Vague
**Avoided by:**
- Specific syllable counts (5-7-5 for haiku)
- Exact rhyme schemes (ABAB CDCD EFEF GG)
- Concrete imagery examples with before/after

### Problem: Missing Context
**Avoided by:**
- Brief explanations before each form
- "Use for" guidance in form selection
- When/why for non-obvious techniques

### Problem: Not Actionable
**Avoided by:**
- Complete poem examples (not fragments)
- Step-by-step revision checklist
- Specific word choice guidance

### Problem: Too Long
**Mitigated by:**
- One example per form (not multiple variations)
- Combined related techniques
- Focused on 90% of use cases

### Problem: No Validation
**Avoided by:**
- Created 4 example poems
- Validated structure against checklist
- Tested with hypothetical task

## Key Differences from Code Templates

**Adaptations made:**

1. **"Project Structure" → "Poetry Forms"**
   - Code templates show directory layout
   - Poetry template shows form structures

2. **Examples are creative, not functional**
   - Code: Must compile/run
   - Poetry: Must demonstrate technique effectively

3. **No installation/setup section**
   - Poetry requires no tools or dependencies
   - Jumped straight to form and technique

4. **Anti-patterns focus on craft, not bugs**
   - Code: Runtime errors, inefficiency
   - Poetry: Weak technique, clichés, unearned emotion

5. **"When Working on" remains checklist format**
   - Same structure adapted to creative process
   - Maintains consistency across templates

## Lessons Learned

1. **Research phase is critical** - Web searches revealed forms I might have forgotten
2. **Question phase provides clarity** - Even with defaults, considering questions shaped scope
3. **Structure first, write second** - Outline prevented rambling
4. **Validation catches issues** - Length check revealed need for condensing (though kept comprehensive)
5. **Examples make it real** - Complete poems demonstrate better than fragments
